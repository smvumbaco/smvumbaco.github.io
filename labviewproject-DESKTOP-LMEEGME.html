Analog Devices: LabVIEW Semiconductor Characterization Interface

At Analog Devices, I served as a Semiconductor Reliability Engineer, where I conducted stress tests on BJTs and MOSFET devices at the wafer and package levels. An important element of this procedure was characterizing the transistors before stressing to obtain baseline pre-test data and ensure that no damage occurred in the development process. Given this requirement, I designed, scripted, and tested a LabVIEW program that interfaced with up to 4 National Instruments source measurement units (SMUs) to quickly bias and characterize semiconductor devices.

When setting up an experiment, users can specify how many SMUs they want to configure as well as what kind of experiment they would like to run. Under a single point measurement, just one value is recorded. With a simple sweep, a basic plot is generated. With a sweep and step experment, multiple sweeps are plotted on the same graph under different biasing conditions. An example of a sweep and step experient would include the output characteristics of a MOSFET at different gate voltage biases. This setup process may be facilitated by importing a previously-used experiment, which auto-populates all the data in the UI.

After defining the experient, users can specify what they want each SMU to source (voltage or current) as well as which SMU is performing a sweep. Naming variables and constants is also available for maximum specification and customization.

Users can also determine the values of the sweep and constant values. The number of measurements is also articulated.

The graph is entirely configurable as well.

Using the test setup articulated above, a simple plot may be configured such as the NPN Gummel plot or nMOS output characteristics graph shown above. This process provides a very clear indication of the health of a transistor very quickly, and experiments may be run as many times as needed. Specific data may also be saved and stored in a simple, readable format with a single button press. Using this LabVIEW system, I was able to significantly reduce the time consumption and cost of characterizing transistors. Much like the Teledyne FLIR current board schematics, the specific code for this project is ADI proprietary information and may not be displayed, but the actual user interface implemented for this project is available and shown above!
